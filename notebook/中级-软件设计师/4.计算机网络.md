# 四、计算机网络

## 1. OSI/RM七层模型

![网络七层模型](../../img/软考/网络七层模型.jpg)
1. 应用层
	- 功能
		- 实现具体的应用功能
2. 表示层
	- 功能
		- 数据的格式与表达、加密、压缩
3. 会话层
	- 功能
		- 建立、管理和终止会话
4. 传输层
	- 功能
		- 端到端的连接
5. 网络层
	- 功能
		- 分组传输和路由选择
	- 主要设备
		- 三层交换机
		- 路由器
	- 特点
		- IP全局广播无法通过此层的设备
6. 数据链路层
	- 功能
		- 传送以帧为单位的信息
	- 主要设备
		- 网桥（连接两个同类型网络）
		- 交换机（多端口网桥，可以连接多个设备）
		- 网卡
7. 物理层
	- 功能
		- 二进制传输
	- 主要设备
		- 中继器（应对信号衰减）
		- 集线器（多线路的中继器）

例题
![网络七层模型例题](../../img/软考/网络七层模型例题.jpg)
- 总结
	- 四个选项一个一个看
	- 同一个局域网下IP全局广播才能通过
	- 跨越3网络层IP全局网络就不能通过了
	- 网桥、交换机是2层数据链路层设备
	- 集线器是1层物理层设备
	- 路由器是3层网络层设备。选B

### 2.3 DNS协议

![DNS协议](../../img/软考/DNS协议.jpg)
- 递归查询
	- 服务器必须回答结果（IP与域名的映射关系）
	- 刨根究底，找到答案，返回结果
	- 负担重，效率低，根域名服务器很少采用
- 迭代查询
	- 服务器收到一次迭代查询回复一次结果，可以不直接回答映射关系结果，可以回答其他DNS服务器地址
	- 提供线索

## 5. IP地址

### 5.1 IP地址分类

![IP地址](../../img/软考/IP地址.jpg)
- 共通特性
	- 主机号全0，一般表示网络地址（该网络的地址）
	- 主机号全1，一般表示广播地址
	- 因此计算网络中可容纳的主机数时，需要减2
- A类地址
	- 前8个位是网络号，后24个位是主机号
	- 首位为0，所以第一段的范围为0-127
	- 同一个网络，可容纳 $2^{24}-2$ 个主机
- B类地址
	- 前16个位时网络号，后16个位是主机号
	- 前两位为10，第一段的范围为128-191
	- 同一个网络，可容纳 $2^{16}-2$ 个主机
- C类地址
	- 前24个位时网络号，后8个位是主机号
	- 前两位为110，第一段的范围为192-223
	- 同一个网络，可容纳 $2^{24}-2$ 个主机
- 无分类地址
	- 网络地址后面会跟斜杠加上一个数字（网络号范围）
	- 例如172.18.129.0/20，指前20位是网络号，后12位是主机号

### 5.2 子网划分

- 子网掩码
	- 概念
		- 用来标明一个IP地址哪些位是主机所在子网（网络号），哪些位是主机号
		- 化为二进制后，为1的部分是网络号，为0的部分是主机号
		- 必须和IP地址一起使用
		- 例如ip地址192.168.2.34 和子网掩码 255.255.255.0，标识网络号是192.168.2.0，后8位是主机号
![子网划分例题1](../../img/软考/子网划分例题1.jpg)
- 例题1：划分子网（部分主机号划为网络号）
	- 例题1总结
		- B类地址，即后16位为主机号。要划分多个子网，说明需要在主机号的左边拿出几位来做子网号
		- $2^4 < 27 < 2^5$，$2^5$ 就是5个1，因此主机号左边要划5位数作为子网号
		- 将得到的结果化为十进制，即得子网掩码
![划分子网例题2](../../img/软考/划分子网例题2.jpg)
- 例题2：划分子网2
	- 例题2总结
		- $2^{10} > 700 - 2 > 2^9$，因此存储700个主机号需要右边有10位的主机号
		- 700 - 2 是因为最低位和最高位一般不用
		- 将得到的结果化为十进制，即得子网掩码
![无分类编址例题](../../img/软考/无分类编址例题.jpg)
- 例题3：无分类编址（无类域间路由）
	- 例题3总结
		- 无分类编址给了20位网络号，C类子网网络号有24位
		- 因此那4位可表达的数就是可划分为C类子网的数量
		- $1111 - 0000 = 2^4 = 16$
